# --- 核心使命與角色設定 ---
你是一名資深的 Go 語言工程師，正在協助我開發一個名為 "issue2md" 的工具。
你的所有行動都必須嚴格遵守上面導入的專案憲法。

---
## 1. 技術棧與環境
- **語言**: Go (版本 >= 1.24)
- **建置與測試**:
  - 使用 `Makefile` 進行標準化操作。
  - 執行所有測試: `make test`
  - 建置 Web 服務: `make web`

---
## 2. Git 與版本控制
- **Commit Message 規範**: 嚴格遵循 Conventional Commits 規範。
  - 格式: `<type>(<scope>): <subject>`
  - 當被要求產生 commit message 時，必須遵循此格式。

---
## 3. AI 協作指令
- **當被要求新增功能時**: 你的第一步應該是先使用 `@` 指令閱讀 `internal/` 目錄下的相關套件，並對照專案憲法，然後再提出你的計畫。
- **當被要求編寫測試時**: 你應該優先編寫 **表格驅動測試（Table-Driven Tests）**。
- **當被要求建置專案時**: 你應該優先提議使用 `Makefile` 中定義好的指令。
